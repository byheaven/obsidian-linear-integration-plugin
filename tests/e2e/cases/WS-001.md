# WS-001 Create Issues From Note In Both Workspaces

## Purpose

Verify that `Create Linear issue from note` works for two enabled workspaces and rewrites the source note into the managed frontmatter-first shape.

## Setup

- Two workspaces enabled in plugin settings.
- Each workspace has at least one configured team.
- Obsidian desktop and CLI are connected to the target vault.

## Steps

1. Create one temporary note per workspace with `@team`, `@status`, `@priority`, and `@label`.
2. Open the note in Obsidian.
3. Drive the create-from-note modal through Obsidian CLI debug mode.
4. Click `Create issue`.
5. Wait for the note to gain `linear_id`.

## Expected Results

- Frontmatter contains `linear_workspace_id`, `linear_id`, `linear_identifier`, `linear_title`, `linear_description`, `linear_status`, `linear_team`, `linear_priority`, and `linear_labels`.
- The description is stored in frontmatter, not in the note body.
- Inline tags are removed from `linear_description`.
- The note body is rewritten into the managed shell.

## Artifacts To Collect

- Modal DOM dump
- before and after note snapshots

## Cleanup

- Delete the temporary notes.
- Permanently delete the remote test issues.
