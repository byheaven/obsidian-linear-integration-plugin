# WS-008 Existing Sync State Sanity Check

## Purpose

Verify that the current multi-workspace sync state still behaves cleanly after the test suite has exercised the plugin.

## Setup

- Two enabled workspaces already configured in the target vault.

## Steps

1. Capture the current inventory of linked notes.
2. Run one more sync.
3. Capture the linked-note inventory again.

## Expected Results

- The test notes remain linked.
- No plugin errors are raised.
- The linked-note inventory does not show unexpected duplicate bindings for the test issues.

## Artifacts To Collect

- before and after linked-note inventory dumps
- `dev:errors`

## Cleanup

- Normal suite cleanup only.
