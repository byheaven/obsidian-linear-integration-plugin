# WS-010 Repair Missing Managed Sections

## Purpose

Verify that linked issue notes missing either `# Document` or `# New Comment` are repaired automatically on the next sync.

## Setup

- At least one linked issue note already exists for each enabled workspace.
- Obsidian desktop and CLI are connected to the target vault.

## Steps

1. Remove the `# Document` section from one linked issue note.
2. Remove the `# New Comment` section from another linked issue note.
3. Run a normal plugin sync.

## Expected Results

- The next sync restores the full managed note shell.
- Each repaired note contains `# Linear Issue`, `# Document`, `# New Comment`, and `# Comments`.
- Existing frontmatter bindings remain intact.

## Artifacts To Collect

- note snapshots before and after repair
- `dev:errors`
