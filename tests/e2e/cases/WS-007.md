# WS-007 Preserve Workspace Isolation

## Purpose

Verify that updates from different workspaces stay on their own notes and do not create duplicates.

## Setup

- WS-001 has already created one linked issue per workspace.

## Steps

1. Update both remote issues with different titles and descriptions.
2. Run sync.
3. Read both local notes.
4. Check linked note uniqueness by `linear_id`.

## Expected Results

- Each note stays bound to its original `linear_workspace_id`.
- Each update lands only on the correct note.
- No duplicate linked notes are created.

## Artifacts To Collect

- post-sync note snapshots
- linked note inventory dump

## Cleanup

- Normal suite cleanup only.
