# WS-003 Push Local Frontmatter Edits Back To Linear

## Purpose

Verify that editing synced frontmatter in Obsidian pushes the changes back to Linear on the next sync.

## Setup

- WS-001 has already created one linked issue per workspace.

## Steps

1. Modify `linear_title`, `linear_description`, `linear_status`, `linear_project`, `linear_priority`, and `linear_labels`.
2. Run sync.
3. Fetch the remote issue again.
4. Read the local note again.

## Expected Results

- Remote issue fields match the local frontmatter edits.
- The next pull rewrites local frontmatter into the canonical normalized values.

## Artifacts To Collect

- note snapshot after local edit
- note snapshot after sync
- remote issue JSON summary

## Cleanup

- Normal suite cleanup only.
