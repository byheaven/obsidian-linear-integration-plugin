# WS-009 Default Team Gates Default Project

## Purpose

Verify that the create-issue modal applies the workspace default team first, then limits the default project fallback to projects that belong to that team.

## Setup

- A workspace has a configured `defaultTeamId`.
- The same workspace has a configured `defaultProjectId` that belongs to that default team.
- At least one other team exists with a different project.

## Steps

1. Open the create-issue modal from a note that does not specify `@team/` or `@project/`.
2. Observe the initial team and project selections.
3. Change the team to a different team.
4. Observe the project selection again.

## Expected Results

- The modal opens with the workspace default team already selected.
- The modal auto-selects the workspace default project only when it belongs to the selected team.
- Switching to a different team clears the previous default project if that project does not belong to the new team.

## Artifacts To Collect

- screenshot of the create modal immediately after it opens
- screenshot after switching teams
- `dev:errors`
