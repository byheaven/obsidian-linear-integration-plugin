# WS-002 Pull Sync Updates Frontmatter Only

## Purpose

Verify that remote Linear edits are pulled into frontmatter while the note body remains the managed shell.

## Setup

- WS-001 has already created one linked issue per workspace.

## Steps

1. Update the remote issue title, description, status, priority, and labels through the Linear API helper.
2. Run plugin sync through Obsidian CLI.
3. Read the local note again.

## Expected Results

- `linear_title`, `linear_description`, `linear_status`, `linear_priority`, and `linear_labels` match the remote issue.
- `linear_last_synced` advances.
- The managed body does not inline the remote description.

## Artifacts To Collect

- post-sync note snapshot
- `dev:errors`
- console errors

## Cleanup

- Normal suite cleanup only.
